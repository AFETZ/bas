## Приоритезация по grant-deadline

Если есть жёсткий дедлайн отчёта по гранту, рекомендуемый порядок:

1. **2.1 Sionna RT** — самый трудоёмкий и научно сильный пункт, начать первым.
   Параллельно с этим Codex/я можем делать 1.7 и 1.8 как короткие задачи.
2. **1.7 LoRa Serial Bridge** — закрывает буквальное расхождение по LoRa.
   Без него ТЗ-таблица показывает «функциональный эквивалент» — слабая
   формулировка для защиты.
3. **1.8 ROS2/MAVROS bridge** — закрывает обязательное требование ROS-стека.
4. **2.4 Ручное управление** — короткий шаг, яркое demo.
5. **2.3 Multi-UAV** — масштабируемость.
6. **2.2 AirSim** — самое тяжёлое, делать после остальных.
